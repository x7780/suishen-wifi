使用说明

开始之前，请仔细阅读

先阅读使用说明，避免因操作不当导致功能损坏。

注意事项
1

设备支持插入手机卡，兼容网通、移动、电信。虚拟号及部分被屏蔽号码可能无法使用，推荐优先使用联通或电信，移动在部分地区受限。

⚠ 如果你感觉手机卡不能使用，不要着急，可以使用家里网络，或者插入电脑直接连接设置。 ⚠ 设备上的盖帽不要拿掉，拿掉会导致手机卡信号变弱，进而出现调制解调器掉驱动的问题，表现为反复重启或联网掉线。手机卡驱动是从安卓移植过来的，本身不稳定，这是设备通病，无法根治。如果不需要使用手机卡，可以停止调制解调器和网桥服务来彻底规避此问题。
2

系统热点与路由器 WiFi 存在冲突，二者只能同时存在一个。连接家庭 WiFi 后热点可能自动断开，建议提前通过以下命令设置固定 IP，避免 IP 丢失。

⚠ 手机卡热点模式下传送文件会很慢或断链，主要原因是使用的安卓驱动，不是原生编译，会出现这样的问题正常，关闭基带和网桥，也可以重新刷linux系统，不用安卓基带就可以解决。
bash
$ nmtui
3

网络基带为设备独有，包含移动网络设置与驱动，每台设备均不相同，丢失无法找回，我这没有备份。

⚠ 一旦丢失无法找回，将导致外部手机卡无法使用。请务必提前备份，切勿删除。
4

如需添加虚拟内存（Swap），可执行以下命令创建 200MB 的交换文件并永久挂载：

bash — 添加虚拟内存 Swap
# 创建 200MB swap 文件
$ sudo dd if=/dev/zero of=/swapfile2 bs=1M count=200
$ sudo chmod 600 /swapfile2
$ sudo mkswap /swapfile2
$ sudo swapon /swapfile2
# 验证是否生效
$ free -h
# 写入 fstab 实现开机自动挂载
$ sudo nano /etc/fstab
# 在文件末尾添加以下一行后保存：
/swapfile2 none swap sw 0 0
5

系统自启动项都存放在 /etc/rc.local 文件中，可通过以下命令查看或编辑，卸载不需要的自启动程序。

bash — 自启动管理
# 查看自启动内容
$ cat /etc/rc.local
# 编辑自启动文件（删除不需要的行即可卸载对应启动项）
$ sudo nano /etc/rc.local
6

设备使用的是 Debian 12，安装软件时若软件源频繁报错，可执行以下命令一键更换为国内镜像源：

bash — 更换软件源
$ bash <(curl -sSL https://linuxmirrors.cn/main.sh)
7

如果内存不够用，或建立虚拟内存也不理想，可以停止蓝牙和 4G 调制解调器服务来释放内存。⚠ 禁用 ModemManager 后将永久无法使用手机卡上网（只能使用wifi网络联网），请谨慎操作。

bash — 释放内存（注意风险）
# 禁用 4G 网络调制解调器（禁用后无法使用手机卡上网）
$ sudo systemctl disable ModemManager
# 禁用蓝牙
$ sudo systemctl disable bluetooth
8

这个没什么用，但是你要自己刷其他系统，切卡遇到了问题，可以尝试一下方法。系统内置两张上网卡：一张是你自己插的手机卡，另一张是自带的 eSIM 卡。⚠ 自带 eSIM 流量非常贵且限速，切勿充值。

以下是切换到外部手机卡的 rc.local 配置，刷机后可参考此配置恢复切卡功能。其中 sim:sel 控制卡槽选择，sim:en 控制卡槽启用，写入 1 表示激活，写入 0 表示关闭。当前配置为：启用卡槽1（外部手机卡），关闭卡槽2（内置 eSIM）。如需切换回 eSIM，将 sim:en 改为 0、sim:en2 改为 1 即可。

bash — /etc/rc.local 切卡配置
# 编辑 rc.local，写入以下内容
$ sudo nano /etc/rc.local

#!/bin/sh -e
# Custom startup script
sleep 10
echo 0 > /sys/class/leds/sim:sel/brightness
echo 1 > /sys/class/leds/sim:en/brightness
echo 0 > /sys/class/leds/sim:sel2/brightness
echo 0 > /sys/class/leds/sim:en2/brightness
modprobe -r qcom-q6v5-mss
modprobe qcom-q6v5-mss
systemctl restart rmtfs
systemctl restart dbus-org.freedesktop.ModemManager1.service
exit 0

# 赋予执行权限
$ sudo chmod +x /etc/rc.local
# 启用并启动 rc-local 服务
$ sudo systemctl enable rc-local
$ sudo systemctl start rc-local
